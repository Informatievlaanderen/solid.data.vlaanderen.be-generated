Entiteiten

Concept

Beschrijving
A SKOS concept can be viewed as a unit of thought, an idea, a notion, a meaning, or (a category of) objects and events, which underlies many knowledge organization systems. However, what constitutes a unit of thought is subjective, and this definition is meant to be suggestive, rather than restrictive. As such, concepts exist in the mind as abstract entities which are independent of the terms used to label them.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: alternative label, definition, in scheme, narrower, notation, note, preferred label, status, top concept of.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
alternative label Language String 0..* An alternative lexical label to a concept. This is especially helpful when assigning labels beyond the one that is preferred for the concept, for instance when synonyms need to be represented.
definition Language String 0..1 Supplies a complete explanation of the intended meaning of a concept.
in scheme Concept Scheme 0..1 Link to the concept scheme that contains the concept.
narrower Concept 0..* This property is used to assert that one concept is narrower in meaning (i.e. more specific) than another.
notation Literal 0..* This property allows a concept to be attached to an RDF typed literalÂ—a literal with an explicit datatype. The datatype of the literal specifies a syntax encoding scheme, which fits the usage of notations in the concerned KOS. The value of the literal is the notation itself.
note Language String 0..* A property for general documentation purposes.
preferred label Language String 1..* A preferred lexical label for a resource.
status Concept 0..1 The status of the concept, e.g. 'in use' or 'deprecated'.
top concept of Concept Scheme 0..1 Indicates that the concept is the most general concept of the concept scheme.

Concept Scheme

Beschrijving
A SKOS concept scheme can be viewed as an aggregation of one or more SKOS concepts. Semantic relationships (links) between those concepts may also be viewed as part of a concept scheme.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: has top concept, status.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
has top concept Concept 0..* This property allows one to link a concept scheme to the (possibly many) most general concepts it contains, in order to provide an efficient access to the entry points of broader/narrower concept hierarchies.
